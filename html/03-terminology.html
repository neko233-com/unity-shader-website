术语与黑话

Unity Shader 中的行话解读 - 让你读代码不再懵逼

📚 为什么需要这个页面？

Unity Shader 代码里有大量的术语、缩写和"黑话"，初学者看到这些会一脸茫然：

  • SV_POSITION 是什么？为什么要加 SV_？
  • Attributes 和 Varyings 是干嘛的？
  • CBUFFER 又是什么鬼？

本页面用通俗易懂的方式解释这些"暗号"，让你从此读 Shader 代码不再犯晕。

🏗️ 数据结构术语

Attributes（属性）

完整名称：Vertex Attributes（顶点属性）

是什么：从 CPU 传给顶点着色器的输入数据

包含信息：顶点位置、法线、UV 坐标、顶点颜色等

类比：就像快递包裹上的"寄件信息"，告诉你这个顶点从哪来、带了什么数据

struct Attributes
{
    float4 positionOS : POSITION;   // 模型空间位置
    float3 normalOS   : NORMAL;     // 模型空间法线
    float2 uv         : TEXCOORD0;  // 纹理坐标
};

Varyings（变量 / 插值数据）

完整名称：Varying Variables（变化的变量）

是什么：从顶点着色器传给片元着色器的中间数据

特点：会在三角形内部自动插值（平滑过渡）

类比：就像"流水线上的半成品"，顶点处理完了传给片元继续加工

struct Varyings
{
    float4 positionCS : SV_POSITION;  // 裁剪空间位置（必须有）
    float2 uv         : TEXCOORD0;    // 传递 UV 给片元着色器
    float3 normalWS   : TEXCOORD1;    // 传递世界空间法线
};

为什么叫 Varying：因为这些值在三角形表面会"变化"（插值）

Attributes vs Varyings 数据流

🏷️ 语义（Semantics）

语义就是变量后面的 : XXXX 部分，告诉 GPU 这个变量是干什么用的。

POSITION

含义：顶点在模型空间的位置

数据来源：来自 Mesh 的顶点数据

使用位置：顶点着色器的输入（Attributes 结构体中）

float4 positionOS : POSITION;  // OS = Object Space

SV_POSITION

完整名称：System Value - Position

SV_ 是什么意思：System Value（系统值），表示这是 GPU 特殊处理的数据

含义：裁剪空间位置（经过 MVP 变换后的位置）

为什么特殊：GPU 需要这个值来判断顶点在不在屏幕上、进行深度测试等

重要：顶点着色器必须输出 SV_POSITION！

// 顶点着色器输出（Varyings 中）
float4 positionCS : SV_POSITION;  // CS = Clip Space

// 顶点着色器中必须赋值
output.positionCS = TransformObjectToHClip(input.positionOS.xyz);

SV_Target

完整名称：System Value - Target

含义：片元着色器的输出颜色目标

作用：告诉 GPU 这个颜色要写到渲染目标（屏幕/纹理）上

可以有多个：SV_Target0, SV_Target1... 用于 MRT（多渲染目标）

half4 frag(Varyings input) : SV_Target
{
    return half4(1, 0, 0, 1);  // 输出红色
}

TEXCOORD0, TEXCOORD1...

含义：纹理坐标（通用插值寄存器）

不只是 UV：虽然叫 TEXCOORD，但可以存任何要插值的数据

数量限制：一般有 TEXCOORD0 ~ TEXCOORD7（8 个）

struct Varyings
{
    float4 positionCS : SV_POSITION;
    float2 uv         : TEXCOORD0;  // 用于传 UV
    float3 normalWS   : TEXCOORD1;  // 用于传法线
    float3 viewDirWS  : TEXCOORD2;  // 用于传视线方向
    // ...可以继续用 TEXCOORD3, 4, 5...
};

常用语义速查表

语义 含义 使用位置
POSITION 模型空间顶点位置 顶点输入
NORMAL 模型空间法线 顶点输入
TANGENT 模型空间切线 顶点输入
COLOR 顶点颜色 顶点输入
SV_POSITION 裁剪空间位置 顶点输出（必须）
SV_Target 输出颜色 片元输出
TEXCOORD0~7 通用插值数据 顶点输出/片元输入

📦 CBUFFER（常量缓冲区）

CBUFFER_START / CBUFFER_END

完整名称：Constant Buffer（常量缓冲区）

是什么：一块传递给 GPU 的数据区域，存放材质参数

为什么要用：为了支持 SRP Batcher，大幅提升渲染性能

规则：所有在 Properties 中声明的变量，都要放在 CBUFFER 里

// Properties 中声明
Properties
{
    _BaseMap ("Base Map", 2D) = "white" {}
    _BaseColor ("Base Color", Color) = (1,1,1,1)
    _Smoothness ("Smoothness", Range(0,1)) = 0.5
}

// HLSL 中对应的 CBUFFER
CBUFFER_START(UnityPerMaterial)
    float4 _BaseMap_ST;   // 纹理的 Tiling 和 Offset
    half4  _BaseColor;    // 颜色
    half   _Smoothness;   // 平滑度
CBUFFER_END

注意：纹理本身（TEXTURE2D）不放在 CBUFFER 里，只有数值类型才放

SRP Batcher 工作原理

🔤 URP 变量命名约定

坐标空间后缀

URP 使用后缀来标识变量所在的坐标空间：

OS Object Space（模型空间）
WS World Space（世界空间）
VS View Space（观察空间）
CS Clip Space（裁剪空间）
TS Tangent Space（切线空间） 
float3 positionOS;  // 模型空间位置
float3 positionWS;  // 世界空间位置
float3 normalWS;    // 世界空间法线
float4 positionCS;  // 裁剪空间位置

下划线前缀 _

规则：在 Properties 中声明的变量，名称以下划线 _ 开头

原因：Unity 的历史约定，区分属性和局部变量

// Properties 声明
_BaseColor ("Base Color", Color) = (1,1,1,1)
_BaseMap ("Base Map", 2D) = "white" {}
_Metallic ("Metallic", Range(0,1)) = 0

// 在 HLSL 中使用
half4 color = _BaseColor;
half4 texColor = SAMPLE_TEXTURE2D(_BaseMap, sampler_BaseMap, uv);

🧩 常用宏定义

TEXTURE2D / SAMPLER

作用：声明纹理和采样器（分离式声明）

为什么分开：多个纹理可以共享一个采样器，节省硬件资源

TEXTURE2D(_BaseMap);           // 声明纹理
SAMPLER(sampler_BaseMap);      // 声明采样器（名字约定：sampler_ + 纹理名）

// 采样时
half4 color = SAMPLE_TEXTURE2D(_BaseMap, sampler_BaseMap, uv);

TRANSFORM_TEX

作用：应用纹理的 Tiling 和 Offset

展开：uv * _MainTex_ST.xy + _MainTex_ST.zw

// _BaseMap_ST 存储了 Tiling (xy) 和 Offset (zw)
float2 uv = TRANSFORM_TEX(input.uv, _BaseMap);

// 等价于
float2 uv = input.uv * _BaseMap_ST.xy + _BaseMap_ST.zw;

TransformObjectToHClip

完整名称：Transform Object Space To Homogeneous Clip Space

作用：将模型空间坐标直接变换到裁剪空间（一步完成 MVP 变换）

// 最常用的变换
output.positionCS = TransformObjectToHClip(input.positionOS.xyz);

// 等价于
output.positionCS = mul(UNITY_MATRIX_MVP, input.positionOS);

📊 数据类型

float vs half vs fixed

类型 精度 范围 用途
float 32 位 很大 位置、UV、精确计算
half 16 位 -60000 ~ 60000 颜色、方向、一般计算
fixed 11 位（已废弃） -2 ~ 2 URP 不再使用

建议：位置用 float，颜色用 half
