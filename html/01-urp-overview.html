URP 概述

理解 Unity Universal Render Pipeline 的核心概念

📚 理论基础

什么是 URP？

URP = Universal Render Pipeline（通用渲染管线）

URP 架构图解

拆解理解

U Universal（通用）

适用于多种平台（PC、手机、主机、VR）— 一套代码，到处运行

R Render（渲染）

把 3D 数据变成 2D 图像的过程 — 就像拍照：3D 的真实世界 → 2D 的照片

P Pipeline（管线）

一系列有序的处理步骤 — 就像工厂流水线：原材料 → 加工 → 成品

📖 URP 的历史演进

Unity 渲染管线演进

1 Built-in Render Pipeline（内置渲染管线）

Unity 最早的渲染系统，功能全面但效率不高，Unity 2022 已标记为弃用

2 SRP (Scriptable Render Pipeline)

Unity 2018 引入的可编程渲染管线架构，URP 和 HDRP 都基于此

3 URP (Universal Render Pipeline)

以前叫 LWRP，2019 年改名。性能优秀，适合大多数项目，本教程的主角

✅ 为什么选择 URP？

🚀 性能优秀

比内置管线快 2-3 倍，针对移动端优化

📱 跨平台

PC、手机、Switch、VR 都能用

🎨 风格化渲染

适合卡通、二次元、像素风格

🔮 官方推荐

Unity 官方推荐的未来主流方向

📖 历史背景与术语

LWRP → URP 改名原因

历史：URP 最初叫 LWRP（Lightweight Render Pipeline）

问题："Lightweight" 让人误解为功能简单

实际：URP 功能强大，只是性能优化更好

改名：2019 年改名为 URP，强调"通用"特性

SRP Batcher

含义：一种批处理优化技术

作用：减少 Draw Call，提升渲染性能

原理：使用相同 Shader 的物体可以合批渲染

要求：Shader 需要兼容 SRP Batcher
